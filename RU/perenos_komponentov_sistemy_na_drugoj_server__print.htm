Перенос компонентов системы на другой сервер

 
Перенос базы данных
Если переносу подлежит база данных:
  • Создайте резервную копию (backup) базы данных системы;
  • Создайте в новом экземпляре MS SQL базу данных с идентичным именем;
  • Выполните задачу восстановления базы из резервной копии (Правый клик по имени базы – Задачи/Восстановить/База данных…);
  • Отредактируйте строку подключения: "C:\Program Files\X-tensive\DPA Host\appsettings.Production.config";
 
 
Примечение: Обратите внимание, если на способ авторизации. Если он изменился - это нужно указать.
При возникновении вопросов - обратитесь в X-tensive.
 
Перенос Host DPA
Если переносу подлежит Host DPA:
  • Установите Host но новый сервер, на шаге «Настройка базы данных» выберите существующую базу;
  • На сервере DPA откройте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config  в строке вида <wcfTransport ipAddress="192.168.0.X" port="45618" />» укажите ip-адрес сервера, на который перенесен Host.
 
Перенос сервера DPA
Если переносу подлежит DPA Сервер:
  • Скопируйте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config;
  • Произведите установку DPA сервера;
  • Скопированный файл DpaServer.config разместите в директории %\ProgramData\X-tensive\DPA\Config\DpaServer.config, заменив файл, автоматически созданный при установке;
  • Добавьте новый сервер к Host (ссылка на раздел с добавлением сервера);
  • Запросите и установите новую лицензию.
