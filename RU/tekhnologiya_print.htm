Технология

 
Технология представляет собой последовательность этапов, состоящих из последовательности операций, где определены:
  • порядок выполнения: очередность, возможность предварительного или параллельного выполнения
  • временные нормы на выполнение,
  • потребность в ресурсах:
          - рабочие центры для выполнения операции;
          - рабочее время (смены);
          - персонал (навыки и допуски/аттестации);
          - сырье;
          - полуфабрикаты;
          - оснастка;
          - инструмент;
          - управляющие программы ЧПУ;
          - расходные материалы;
          - транспорт;
          - места складирования;
          - контрольно измерительный инструмент;
          - документация.
 
Технология подразумевает этапы - последовательность операций выполняемая на одном рабочем центре. Обычно рассчитана на единичное, групповое, объемное, порционное изготовление. Технология - имеет несколько версий, в один момент времени может быть несколько активных технологий. На заказ может быть назначена только активная технология.
Для работы в модуле "Технология" у пользователя должны быть настроены соответствующие права доступа к функциям (редактирование технологии для управления технологией) и заполнены соответствующие справочники.

 Редактирование технологии
 
Элементами технологии являются этапы и операции.  Этап – группа операций по производству изделия, в частном случае выполняемая на одном рабочем центре. Технология может содержать один и более этапов. Каждый этап может содержать один и более этапов. Операции подразделяются на категории и типы. 
По категориям операции подразделяются на:
• Инструкция – отображение инструкций для оператора;
• Работа по УП – проверка активной УП на стойке на соответствие технологии, предоставление доступа к хранилищу УП;
• Контроль качества – отображение инструкций по контролю качества для оператора;
• Ввод данных – ввод оператором специфицированных данных.
Индикация категорий реализована в виде пиктограмм.
 
Технологии. Редактор
 
По типу операции относятся к партии изделий (наладка-партия, завершающая-партия) или к конкретному изделию (наладка-штука, основная-штука, завершающая-штука). Индикации типа реализована в виде уровней вложенности и цвета (операции цикла изготовления одного изделия отмечены синим).  Созданная технология имеет стандартную структуру, которую можно изменить: 1 этап, 2 партионные операции и 4 операции по каждой единице изделия. Управление списком элементов осуществляется кнопками . Изменять порядок операций можно перетаскиванием. 
Для каждой операции может быть задана норма длительности. Длительность этапа равна сумме нормативной длительности операций.  В верхней части окна расположены вкладки, относящиеся к технологии в целом:
•  «Редактор» - основная рабочая область;
•  «Шаблон» - для отображения исходного xml-шаблона импортированных извне технологий ;
•  «Медиа файлы» - медиа файлы по технологии в целом, загруженные файлы будут доступны оператору в процессе работы по заданию при переходе к связной операции. Файлы могут быть обязательными к просмотру, в этом случае оператор не сможет избежать их просмотра;
•  «История» - история редактирования технологии.
