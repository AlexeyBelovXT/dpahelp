Масштабирование

Для проектов, предназначенных для мониторинга большого количества единиц оборудования, необходимо устанавливать компоненты системы на отдельные сервера с разделением по ролям:
  • WEB-сервер (DPA Host);
  • Сервер баз данных: реляционная и Real-Time (БД DPA и индикаторы DPA);
  • DPA-сервер.
Подключение DPA-серверов осуществляется посредством коммутатора/маршрутизатора. Количество DPA-серверов зависит от количества станков, их расположения и объема трафика. К одному DPA-серверу могут быть подключены до 70 станков. Снимаемые сигналы со станков конвертируются и передаются посредством интерфейсов Ethernet, в протоколах TCP/IP и Modbus RTU на сервер DPA.
При установке компонентов системы на отдельные сервера (подключение до 1000 единиц оборудования), необходим следующий набор компонентов:
  • Операционная система: Windows Server 2016 Standard x64;
  • СУБД: Microsoft SQL Server 2017 Standard x64;
  • Процессор:  Xeon, частота, 3.7/4.1 ГГц, 16 ядер/32 потока;
  • Оперативная память: 16 Гб;
  • Жесткий диск: SATA RAID 0, 7200RPM, 2 Тб – 2 шт.;
  • SSD-накопитель: SATA, 1 Тб, максимальная скорость записи 2 Гб/с, максимальная скорость чтения 2 Гб/с – 2 шт.;
  • Ethernet – 10Гбит.
     
    Примечание: При полноценном развёртывании, как правило DPA Server - это цеховая сеть (может быть несколько), а DPA Host - это офисная сеть. Между DPA Server и DPA Host должна быть корректно настроена маршрутизация.
