Безопасность. Пользователи, группы, права доступа

 
Для того чтобы обеспечить разделение прав доступа к отдельным функциям системы между специалистами предприятия, а так же контролировать время работы в системе и вести корректно историю ввода данных пользователями рекомендуется создавать персональные учетные записи для всех специалистов, имеющих доступ к DPA.
По умолчанию создана одна учетная запись с установленным способом входа типа "логин/пароль". Не рекомендуется сообщать кому-либо указанные данные для входа.
В случае если вы воспользовались опцией "Предустановленные данные" в системе будет создано несколько пользователей без способов входа. Эти пользователи созданы для целей демонстрации возможностей системы и, в качестве примера настройки политики полномочий.
 
После этого можно переходить к построению Структуры предприятия и заполнению рабочих Справочников.
