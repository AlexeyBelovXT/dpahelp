Move components to other server

 
Transfer database to other server
 
Если переносу подлежит база данных:
 
  • Create the backup copy of the database by management console (in case of MS SQL it is SQL Management Studio);
  • Create on a new instance of MS SQL database with an identical name;
  • Carry out a task of base recovery from the backup;
  • Edit a line of connection: "C:\Program Files\X-tensive\DPA Host\appsettings.Production.config";
Note: Pay attention to a way of authorization. if it changed - it needs to be specified.
At emergence of questions - address specialists X-tensive.
 
 
Transfer of DPA Host
 
  • Install Host on the new server, on a step "Setup of the database" select the existing base;
  • On the DPA Server open the file  %\ProgramData\x-tensive\DPA\Config\DpaServer.config". In the line with <wcfTransport ipAddress= "192.168.0.X" of port= "45618"/> specify the IP address of the new server.
 
 
Transfer of DPA Server
 
  • Copy file %\ProgramData\X-tensive\DPA\Config\DpaServer.config;
  • Install DPA server;
  • The copied DpaServer.config file place in a %\ProgramData\X-tensive\DPA\Config\DpaServer.config, having replaced the file which is automatically created at installation;
  • Add the new server to DPA Host;
  • Request and set the new license.
 
Transfer of DPA Server и DPA Host (all-in-one)
 
  • Check correct time on the server where you make installation;
  • On the new server install the DPA Server and DPA Host;
  • Through DPA Host request the new license;
  • Stop services DPA Host and DPA Server;
  • Copy file %\ProgramData\x-tensive\DPA\Config\DpaServer.config to the new server;
  • Stop service of services DPA Host and DPA Server on that server from where you make transfer of a system;
  • Start service of service DPA Server on the new server;
  • Transfer a database backup to the new server;
  • Start DPA Host on the new server;
  • Set the license for DPA Host requested earlier on the new server;
