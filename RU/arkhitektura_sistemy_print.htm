Архитектура системы

 
DPA – это программный комплекс, включающий следующие компоненты:
 
  • DPA Server – отвечает за непосредственное взаимодействие со оборудованием мониторинга, он производит сбор данных, осуществляет их первичную обработку и передает в Host;
  • DPA Host – агрегирует и обрабатывает данные с DPA-серверов, обрабатывает их, записывает их в БД DPA и файлы индикаторов DPA, а так же отвечает за работу пользовательского интерфейса;
  • БД DPA – реляционная база данных, предназначенная для хранения системных таблиц, справочной информации, значимых машинных данных и пользовательских данных;
  • Индикаторы DPA - база данных реального времени, предназначенная для оптимизации процессов записи и чтения часто изменяемых машинных данных, а так же оптимизации использования ресурсов сервера.
 
Все компоненты системы могут быть размещены на отдельном сервере (физическом или виртуальном). Требования к серверному оборудования обусловлены:
  • Компонентами которые развернуты на конкретной машине;
  • Уровнем распределённости;
  • Количеством оборудования мониторинга.
 
DPA Server работает с файлом конфигурации мониторинга, который содержит информацию об ip-адресах серверных компонентов (DPA Server и DPA Host),  ip-адресах оборудования мониторинга, а так же параметры чтения машинных данных. Файл конфигурации хранится на сервере DPA, изменения в файл конфигурации могут быть внесены напрямую (авторизованными специалистами DPA) или через графический интерфейс DPA Host (администраторами экземпляра системы), к которому подключен DPA Server. Прочие настройки системы, в том числе настойки использования  Индикаторов DPA хранятся в БД DPA.  DPA Host - единственных компонент системы, который имеет графический интерфейс пользователя.
