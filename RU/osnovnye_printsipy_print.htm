Основные принципы

1. Для каждого типа данных выбирается своя “мастер-система” (главная).
 
Мастер-система гарантирует корректность и актуальность данных соответствующего типа. Информация “создаётся”  и “редактируется” (изменяется) только в мастер-системе.
Остальные системы выполняют синхронизацию с мастер-системой по соответствующему типу данных. Работа со значениями строго в режиме “только чтение”.
 
Для каждого типа данных может быть выбрана своя мастер-система:
  • Номенклатура продукции – ERP
  • Заказы – ERP
  • Состав изделия – PLM
  • Управляющие программы – PLM
  • Оперативный план – APS
  • Сменно-суточные задания – APS
  • Факт выполнения сменно-суточных заданий – DPA
  • Машинные данные – DPA
и т.д.
 
2. Способы интеграции
 
API, например, web-сервисы или REST-API
Позволяет организовать взаимодействие по принципу запрос-ответ.
Из плюсов – обмен данными в режиме, близком к реальному времени.
Из минусов – низкая отказоустойчивость. Если одна из систем не доступна в данный момент времени, запросы обработаны не будут. В таких случаях требуется предусматривать механизмы повторной обработки запросов.
 
Шины данных, например, MSMQ
Очереди. Взаимодействие по принципу “первый пришел - первый вышел”.
Есть гарантия доставки, есть детерминированный порядок обработки сообщений. Может потребовать определенных усилий администрирования.
Дает хорошую отказоустойчивость и масштабируемость.
Доступны прочие шины данных, например, RabbitMQ, NServiceBUS и другие - могут включать встроенные средства маршрутизации, ведения журналов, диагностики.
 
Интеграционные базы данных
Дает гарантию доставки. Есть транзакционность. Можно организовать взаимодействие сразу нескольких информационных систем.
Из минусов – требуется разработка и программирование логики работы с конкретной синхронизационной базой данных. Поддержка и сопровождение.
 
Общее файловое хранилище
Обмен файлами. Например - XML. Обеспечивает стандартный способ кодирования содержания, предоставляет гибкий способ создания структур данных.
Не гарантирует порядка обработки сообщений. Но позволяет организовать интеграцию между системами принципиально разных платформ (windows- и linux-системы)
