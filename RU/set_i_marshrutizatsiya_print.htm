Сеть и маршрутизация

 
Схематичное описание сетевой структуры
 
  • DPA Server:
    Производит первичную обработку данных и отсылает пакеты DPA Host, загружает драйвера под конкретные типы ЧПУ и отвечает за непосредственное взаимодействие со станками для сбора и передачи данных.
    DPA Server взаимодействует непосредственно со станками и должен находиться в цеховой сети. К одному DPA Server могут быть подключены 70 -100 станков.
    Примечание: Количество станков зависит от объема трафика, количество DPA Server зависит от количества станков, их расположения и объема трафика. DPA Server отправляет на DPA Host данные по подписке (только в случае, если данные изменяются). Таким образом значительно снижается нагрузка на канал передачи данных при использовании распределенной конфигурации.
 
  • DPA Host:
Производит сбор и протоколирование данных с DPA Server, отвечает за работу пользовательского интерфейса. Для повышения надежности данных могут быть использованы кластерные решения на базе стандартных функция MS Windows Server.
 
  • БД DPA:
Для повышения надежности данных могут быть использованы кластерные решения на базе стандартных функция MS SQL Server.
 
Требования к маршрутизации:
С сервера DPA должен быть доступ к подключаемым стойкам. Номера портов (уточняются на первом этапе пилотного проекта) зависят от типа ЧПУ и могут зависеть от версии человеко-машинного интерфейса и/или программного обеспечения, установленного на стойке.

Последствия потери связи:
  • станок-сервер-DPA - потеря данных за период неполадок сети;
  • сервер-DPA-Host - риск потри данных при переполнении ресурсов сервера (сервер копит пакеты данных для отправки на Host в оперативной памяти);
  • При потери связи DPA-Host-БД DPA система потеряет работоспособность.
