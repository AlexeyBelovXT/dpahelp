Before installation

 
Installation of DPA requires the productive server with the x64 Microsoft Windows operating system (OS).
 
Check network settings. The DPA components can be installed on several machines. Make sure that all servers are connected to one local area network, have necessary settings.
 
Prepare the databases server. The detailed installation instruction of MS SQL it is possible to look on Microsoft website.
Pay attention that the control console of the SQL server does not go in a set. The installer needs to be loaded from the official site.
The console can be deployed both on the SQL server, and other machine (for example the administrator's PC). Make sure that it is possible to connect via the console to the SQL server.
 
Requirements to the additional software:
 NET Framework 4.6.2 (https://www.microsoft.com/ru-ru/download/details.aspx?id=53344);
 
During the DPA installation the database will be created and configured and also services of applications are started. The described operations can be performed on behalf of different account types:
· Localsystem. The account which is built in the operating system. By default has the highest privileges on the server;
· The integrated account (Windows). Installation of a system, creation and setup of base and also services can is launched on behalf of the account created on the server or in the domain Active Directory;
· User of SQL. The account is created via SQL server management console. Such account has an access only to components and services of the corresponding SQL server.   
 
We do not recommend to use the built-in well-known accounts. In addition to the known names, the built-in accounts have excessive (over necessary) rights and accesses which are not required for DPA work. It is safer to create separate accounts and to use them for installation the DPA component.
 
Make sure that the corresponding users possess necessary and sufficient (not moreover) rights.
