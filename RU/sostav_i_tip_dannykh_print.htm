Состав и тип данных

DPA  поддерживает импорт данных через передачу xml-файлов. В теле файла описывается одно или несколько задач импорта. Задачи можно классифицировать в четыре основных группы: только создание объекта, создание или обновление объекта, только обновление объекта, удаление объекта. Дополнительно к этим группам задач, существуют задачи для управления расписаниями работы и управлениями заданиями.
Следующие сущности могут быть импортированы в DPA:
  • Справочник “Бригады”
  • Справочник “Тип циклов УП”
  • Справочник “Причины простоя”
  • Справочник “Типы причин простоя”
  • Справочник “Типы примечания”
  • Справочник “Причины отклонения”
  • Справочник “Должности”
  • Справочник “Типы обработки”
  • Справочник “Типы изделия”
  • Справочник “Тип управляющей программы”
  • Справочник “Наименования смен”
  • Справочник “Причины приостановки”
  • Справочник “Группы рабочих центров”
  • Справочник “Шаблоны расписаний”
  • Графики работы рабочих центров
  • Управляющие программы и их версии
  • Изделия и их версии
  • Технологии
  • Сменно-суточные задания и управление ими
     
    Тип данных и передаваемые значения
  • Бригада (наименование, бригадир, список персонала входящего в бригаду)
  • Тип циклов УП (наименование)
  • Причина простоя (наименование, цвет, тип причины простоя, категория, признаки доступности для установки в аналитиках и операторе)
  • Тип причин простоя (наименование)
  • Тип примечания (наименование)
  • Причин отклонения (наименование, цвет)
  • Должность (наименование)
  • Тип обработки (наименование)
  • Тип изделия (наименование)
  • Тип программы (наименование)
  • Наименование смены (наименование, цвет)
  • Причина приостановки (наименование, цвет)
  • Группа рабочих центров (наименование, список оборудования)
  • Шаблон расписания (наименование, тип, список интервалов)
  • Управляющая программа (наименование, тип обработки, тип программы, категория, подразделение, комментарий)
  • Версия управляющей программы (статус, номер версии, описание, количество выпускаемых изделий, время наладки, время выполнения, файл, список оборудования)
  • Изделие (код, наименование, категория, тип, подразделение, комментарий)
  • Версия изделия (статус, номер версии, описание, список оборудования, список компонентов)
  • Технология (наименование, описание, период активности, статус, производимое изделие, список этапов, список медиа-файлов)
  • Этап технологии (наименование, описание, список операций, список оборудования, список медиа-файлов)
  • Технологическая операция (наименование, описание, длительность, тип, список медиа-файлов)
  • Сменно-суточные задания (идентификатор из внешней системы, описание, комментарий, статус, оборудование, плановое и актуальное время начала/окончания, список медиа-файлов, список ссылок на документацию)
  • Производство (заказ, номер партии, тип, плановое количество, этап технологии, версия изделия)
  • ТОиР (тип, причина)
 
Полную информацию о структуре xml-файлов можно получить из xsd-схем, размещённых на нашем ftp (ftp://ftp.x-tensive.com/Public/Downloads/DPA.Schemas/)
