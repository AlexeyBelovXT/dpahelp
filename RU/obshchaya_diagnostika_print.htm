Общая диагностика

 
1. Если требования к подготовке сервера не были соблюдены, процесс установки не может быть завершен корректно.
В случае, если в процессе установки возникли ошибки, например, связанные с недостаточностью полномочий, финальный экран будет содержать ссылки на лог-файлы установки – направьте файлы в службу поддержки X-tensive.
                                                          Мастер установки. Отмена установки
 
В случае, если ошибки настроек/сбой работы SQL позволили компонентам системы стартовать, но не позволили подключиться к базе данных при запуске системы вместо окна авторизации будет приведено описание проблемы, которое необходимо направить в службу поддержки X-tensive.
                                                                                                     Ошибка подключения к базе данных (пример)
 
2. Перед обращением в X-tensive необходимо удостоверится, что все требования соблюдены, необходимо запустить SQL Server Management Studio выполнить ряд проверок.
Для новых баз данных:
·   Удалось ли создать базы данных с именами, выборными вами на шаге «Настройка базы данных»?
·   Удалось ли создать таблицы?
Для существующих баз данных:
·   В настройках безопасности баз данных присутствуют учетные записи выбранных пользователей?
 
3. В случае, если возникли проблемы с запуском отдельных компонентов системы, запустите приложение «Службы» и проверьте статус следующих служб:
– DPA Server;
– DPA Host;
В случае, если один или несколько компонентов находятся в состоянии отличном от «Выполняется», запустите их в указанном порядке.
Перед обращением в X-tensive необходимо выполнить ряд проверок:
 
  • Удостоверьтесь, что пользователь, выбранный для запуска служб добавлен в соответствующую группу доступа (Локальные политики безопасности\Локальные политики\Назначение прав пользователя\Вход в качестве службы).
                                                                                                                                                  Локальные политики безопасности (пример)
 
  • Запустите командную строку от имени администратора и удостоверьтесь, что порт выбранный для запуска Хост на этапе «Конфигурация DPA Host» зарезервирован для запуска от имени пользователя (netsh http show urlacl):
                                                                                                         Конфигурация сетевых параметров (пример)
В ходе обращения сообщите в X-tensive о результатах проверок. 
 
4. В случае если экземпляр базы данных, используемый системой DPAHost развернуты на одном сервере, рекомендуется установить зависимость между службами.
Запустите приложение «Службы» и скопируйте название службы SQL Sever’a, затем вызовите консоль в режиме администратора и выполните команду вида «sc config "DpaHost" depend= "MSSQLSERVER".
