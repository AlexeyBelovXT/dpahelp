Настройки системы

На панели навигации нажмите на кнопку , и в выпадающем списке выберите пункт меню «Настройка системы».
Главное меню. Стартовая страница
 
Данный раздел предназначен для установки пользовательских настроек системы.
В таблице ниже приведен полный перечень настроек и пояснений:
№ п/п
Раздел/Подраздел
Назначение
1.
Интеграция
Настройки интеграции системы
1.1.
Импорт
Директива пути, ссылающегося на местоположение каталога файловой системы для импортируемых данных
Активация импорта файлов
Формирование ответов на внешние запросы
1.2.
Экспорт
Директива пути, ссылающегося на местоположение каталога файловой системы для выгрузки экспортируемых данных
Добавление функции Url очереди MSMQ для выгрузки сообщений, позволяющих обеспечить поддержку асинхронных событий системы обмена сообщениям
Активация выгрузки контрольно-измерительных данных системы Renishaw
Директива пути, ссылающегося на местоположение каталога файловой системы для выгрузки экспортируемых данных системы Renishaw
1.3.
Выгрузка заказов
Директива пути, ссылающегося на местоположение каталога файловой системы для выгрузки данных заказов
Активация выгрузки данных о старте заказа
Активация выгрузки данных об окончании заказа
Активация выгрузки детальных данных о выполнении заказа
1.4.
Выгрузка ТОиР
Активация выгрузки данных о ТОиР
Директива пути, ссылающегося на местоположение каталога файловой системы для выгрузки данных ТОиР
Добавление функции Url очереди MSMQ для выгрузки сообщений, позволяющих обеспечить поддержку асинхронных событий системы обмена сообщениям
1.5.
Выгрузка простоев
Активация выгрузки данных о простоях оборудования
Директива пути, ссылающегося на местоположение каталога файловой системы для выгрузки данных простоев оборудования
Добавление функции Url очереди MSMQ для выгрузки сообщений, позволяющих обеспечить поддержку асинхронных событий системы обмена сообщениям
2.
Учёт простоев
Настройки параметров учёта простоев: длительность простоев и классификация состояний простоев
2.1
Автоматически классифицировать короткие простои
Активация автоматической классификации коротких простоев
2.2
Длительность короткого простоя
Максимальная длительность простоя, который будет автоклассифицирован системой, как короткий (работает только при активированной автоклассифкации) 
2.3
Игнорирование коротких простоев в модуле оператор
Позволяет активировать скрытие коротких простоев для модуля "Оператор". При активированной опции все простои длительность короче установленной будут скрыты от оператора и объединены в сегмент «Короткие простои», автоклассификация при этом не происходит
2.4
Длительность короткого простоя в модуле оператор
Максимальная длительность простоя, который будет определен как короткий при отображении для оператора
2.5
Автоматически классифицировать плановые простои
Активация автоматической классификации плановых простоев (ТОиР и Нерабочее время)
2.6
Классифицировать простой состояния 'Аварийный стоп'
Активация возможности классификации состояний "Аварийный стоп"
2.7
Классифицировать простой состояния 'Наладка'
Активация возможности классификации состояний "Наладка"
3.
Отклонения от нормы выполнения
Настройки параметров отклонений: длительность отклонений и классификацию отклонений от нормы
3.1
Классификация отклонений от нормы оператором
Активация возможности классификации отклонений от нормы длительности операций при работе по технологии
3.2
Процент отклонения
Предоставляет возможность задать процент возможного отклонения от нормы
3.3
Длительность отклонения (секунды)
Предоставляет возможность задать длительность отклонения от нормы
4.
Хранение данных
Настройки параметров хранения данных: путь хранения лога индикаторов, данные вибромониторинга и медиа файлы
4.1
Лог индикаторов
Директива пути хранения индикаторов DPA на сервере системы
4.2
Данные вибромониторинга
Директива пути хранения данных выбромониторинга на сервере системы
4.3
Медиа файлы
Директива пути хранения медиафайлов технологии на сервере системы
5.
Электронная почта
Пользовательские настройки (авторизация и хранение сессии), настройка входящих/исходящих параметров для почты
5.1
Отображаемое имя
Обозначение имени, которое будет отображаться выбранному для отправки почтовому адресу 
5.2
Пароль
Указание пароля доступа к выбранному для отправки почтовому адресу 
5.3
Пользователь
Позволяет указать почтовый адрес, который будет использоваться для отправки сообщений
5.4
SMTP-порт
Предоставляет возможность задать порт почтового сервера
5.5
SMTP-сервер
Позволяет указать адрес почтового сервера
5.6
Защищённое соединение
Настройка параметров шифрования передаваемой информации, с помощью преобразования данных через защищённый протокол
5.7
Отключить отправку электронной почты
Активация функции отправки электронной почты выбранному адресату
6.
СМС
Настройки сервиса коротких сообщений
6.1.
Провайдер
Настройка параметров протокола провайдера для доставки сообщений
6.2.
GSM Modem провайдер
Настройка параметров СОМ-порта, Baud rate и таймаута
6.3.
HTTP провайдер
Позволяет указать URL отправки сообщений, имя параметра: номера телефона, текста сообщений и подписи отправителя
6.4.
SMPP провайдер
Настройка параметров протокола SMPP
6.5.
SMTP провайдер
Настройка параметров протокола SMТP
7.
Диспетчеризация
Настройки управления списками пользователей/группой пользователей и настройки доступа
7.1
Простой
Активация вывода сообщений о простоях оборудования в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.2
Авария
Активация вывода сообщений об аварийных ситуациях оборудования в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.3
УП
Активация вывода сообщений о статусах управляющей программы в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.4
Выпуск
Активация вывода сообщений о выпуске изделий в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий о выпуске изделий в модуль «Диспетчеризация»
7.5
Результат сравнения текста УП
Активация вывода сообщений о результатах сравнений текста управляющих программ в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.6
Сообщение
Активация вывода сообщений об ошибке/аварии/предупреждении/информации в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.7
Сигнал
Активация вывода сигналов в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.8
Запрос оператора
Активация вывода запросов от оператора в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.9
Технологический цикл
Активация вывода событий по операциям технологического цикла в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
7.10
Заказ
Активация вывода сообщений о статусах заказов в модуль «Диспетчеризация» и возможность выбора пользователя/группы пользователей, для отслеживания поступающих событий
