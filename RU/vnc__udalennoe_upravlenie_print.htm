VNC. Удаленное управление

 
 
Начиная с версии 4.7 в DPA появилась функция подключения к станку через систему удаленного доступа VNC (Virtual Network Computing).
Если на станке установлен VNC сервер, то с помощью встроенной в DPA программы-клиента вы можете просматривать Рабочий стол, либо удаленно управлять станком с помощью клавиатуры и мыши.
 
Примечание. Если не установлен VNC сервер.
Существует несколько VNC серверов. Мы протестировали работу со следующими бесплатными решениями:
https://www.uvnc.com/downloads/ultravnc/  
https://www.tightvnc.com/download.php 
Обратите внимание, что для скачивания предлагают 32битную и 64битную версии ПО. Выберите ту, которая подходит для станка.
Затрудняетесь с выбором, загружайте и устанавливайте 32битную версию.
 
 
Для подключения к станку необходимо изменить соответствующий конфигурационный файл на DPA server.
Перед этим вы должны узнать значение сетевых настроек подключения, которые будете вносить в конфигурацию.
 
Запрашиваем у администратора информацию по станку.
 
·     IP-адрес станка.
·     Порт подключения к VNC сервису станка. Обычно это 5900 (или 5800), но лучше посмотреть настройки непосредственно на станке.
·     Пароль на просмотр изображения со станка
·     Пароль на управление станком
Примечание: Пароли могут быть пустыми («»).
 
 
Меняем конфигурационный файл на DPA server
 
Рассмотрим вариант, когда у вас есть следующая информация:
IP-адрес и порт “10.0.0.183" и "5900”
пароль на управление “@dm1n”
 
Теперь можно приступать к редактированию файла, который располагается на сервере с установленным DPA Host.
 
1)     Открываем папку “C:\ProgramData\X-tensive\DPA VNC”
Примечание: по-умолчанию папка C:\ProgramData является скрытой, поэтому вы не увидите ее через Проводник. Рекомендуем в адресной строке сразу указать полный путь до папки и нажать Enter
Либо выбрать Пуск-> Выполнить (комбинация клавиш WIN+R) и указать путь до папки, после чего нажать Enter
2)     В папке редактируем файл DpaVnc.config
В этом файле содержатся настройки станков, к которым можно подключаться по VNC
Структура файла следующая:
{
"Vnc": {
"Equipment1":{...},
....
"EquipmentN":{...}
}
}
Первые 2 строки и последние 2 строки содержат обязательные настройки.
"Equipment?": - это уникальный идентификатор каждого станка, к которому есть подключение через VNC
сразу после идентификатора идет группа настроек конкретного станка {...}
Между идентификатором и группой настроек - "двоеточие"
Если станков несколько, то настройки разделяются запятыми (после закрывающей скобки } )
Название и значение параметра указываются в двойных кавычках.
Примечание: Сколько есть открывающих скобок {, столько должно быть закрывающих }
 
По каждому станку необходимо указать следующие параметры:
  • Уникальный идентификатор, не повторяющийся внутри файла. По сути это идентификатор группы настроек станка.
  • «name»: это наименование станка, как оно будет указано на странице VNC подключения  (http://dpa_server:9000)
  • «url»: адрес подключения, указывается в формате «IP-адрес/порт»
  • «view»: пароль для просмотра экрана станка.
  • «control»: пароль для удаленного управления станком
    Примечание: Если пароля нет, то пустое значение указывается кавычками без символов внутри:   «»
 
Пример конфигурации одного станка:
"Equipment1":{
“name”: “TEST”
“url”: “10.0.0.183/5900”
“view”: “”
“control”: “@dm1n”
}
 
Примечание: если необходимо отключить управление, то вместо удаления параметра достаточно в начале строки поставить символы комментария //
Тогда параметр не будет учитываться при организации подключения. Например:
//“control”: “@dm1n”
 
Для двух станков с идентификаторами "Equipment1", "Equipment2" файл будет выглядеть так:
 
{
"Vnc": {
 
"Equipment1":{
“name”: “TEST”
“url”: “10.0.0.183/5900”
“view”: “”
“control”: “@dm1n”},
 
"Equipment2":{
“name”: “TEST1”
“url”: “10.0.0.173/5900”
“view”: “FGghER45”
“control”: “admin”}
 
}
}
 
3)     Сохраняем файл
 
 
Проверяем результат
Переходим по адресу http://имя_DPA_host:9000, вместо имя_DPA_host укажите полное имя или IP-адрес сервера, на котором установлен DPA Host.
Появится примерно такая страница:
Это список станков, как указано в конфигурационном файле. Рядом ссылки с разрешенными действиями, к которым был указан пароль.
Если указали пароли как для Просмотра, так и для Управление, то в строке будут две ссылки "View" и "Control".
