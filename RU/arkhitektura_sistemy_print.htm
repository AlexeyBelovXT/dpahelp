Архитектура системы

 
DPA – это программный комплекс, включающий следующие компоненты:
 
  • DPA Server – отвечает за непосредственное взаимодействие со оборудованием мониторинга, он производит сбор данных, осуществляет их первичную обработку и передает в Host;
  • DPA Host – агрегирует и обрабатывает данные с DPA-серверов, обрабатывает их, записывает их в БД DPA и файлы индикаторов DPA, а так же отвечает за работу пользовательского интерфейса;
  • БД DPA – реляционная база данных, предназначенная для хранения системных таблиц, справочной информации, значимых машинных данных и пользовательских данных;
  • Индикаторы DPA - база данных реального времени, предназначенная для оптимизации процессов записи и чтения часто изменяемых машинных данных, а так же оптимизации использования ресурсов сервера.
  • Виртуальный драйвер - программное обеспечение, позволяющее выполнять код по управляющей программе и эмулировать, поступающие сигналы с оборудования (состояние станка, ошибки, режимы работы, сообщения и т.п).
  • Агент - программное обеспечение, обеспечивающее логирование и передачу данных о работе и состоянии оборудования на сервер;
 
Виды структур аппаратных платформ:
 
  • Все компоненты системы могут быть размещены на отдельных серверах (физических или виртуальных) для повышения надежности системы. При этом к системе может быть подключено несколько серверов DPA, к каждому из которых может быть подключено до 70 станков. В случае установки компонентов на независимые сервера, ключевыми требованием к серверу базы данных - в проектах с распределенной аппаратной платформой, является скорость передачи данных (оказывает прямое влияние на быстродействие системы) и необходимость обеспечения корректной маршрутизации пакетов. 
 
  • Все программные компоненты могут быть установлены на один производительный сервер (данная конфигурация рекомендуется для реализации в рамках малых и стартовых проектов).
     
    Примечание: DPA Server работает с файлом конфигурации мониторинга, который содержит информацию об IP-адресах серверных компонентов (DPA Server и DPA Host),  IP-адресах оборудования мониторинга, а так же параметры чтения машинных данных. Файл конфигурации хранится на сервере DPA, изменения в файл конфигурации могут быть внесены напрямую (авторизованными специалистами DPA) или через графический интерфейс DPA Host (администраторами экземпляра системы), к которому подключен DPA Server. Прочие настройки системы, в том числе настойки использования индикаторов DPA хранятся в БД DPA.  DPA Host - единственных компонент системы, который имеет графический интерфейс пользователя.
