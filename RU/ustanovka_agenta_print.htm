Установка агента

 
 
Без установленного STEP7 станок не сможет сам отдавать необходимую информацию. Поэтому необходимо установить Агент DPA, который будет подключаться к служебным сервисам и собирать с них информацию.
Примечание: На PCU станка есть три сервиса ncdde, mbdde, dhserver. Это службы, которые в виде подписки скидывают агенту все изменения на индикаторах станка.
Если сервисы перестают работать, то агент перестает получать данные.
 
 
Где взять установочные файлы?
Новые версии агентов, так же, как и системы DPA, можно скачать с FTP сайта
ftp://ftp.x-tensive.com/Public/DPA%20Agents/
текущая версия: dpa.agent(#49).install.exe
C HMI нет прямого доступа в интернет, поэтому необходимо выложить файл на локальный ресурс. Подойдет папка на DPA Server, поскольку станки должны иметь к нему доступ по сети. Далее скопировать файл на локальный диск HMI.
 
Просим системного администратора создать папку общего доступа для хранения установочных файлов DPA. Например папка Agents. Администратор должен настроить права доступа таким образом, чтобы мы могли читать данные не зависимо от пользователя, которым зашли на PCU. Т.е. доступ должен быть открыт как для доменных, так и для локальных пользователей. Например создана папка \\имяDPAserver\АгентыDPA
 
1) Чтобы попасть в эту папку, открываем на PCU Проводник и вбиваем адрес, указанный администратором.
2) Находим самый свежий, по порядковому номеру, агент и перетаскиваем к себе на Рабочий стол
 
Установка агента
Агент не требует дополнительные компоненты. Поэтому сразу можно запускать установку.
Важный момент: если агент уже есть в системе, то установщик вместо обновления может только удалить существующую версию. Потребуется повторный запуск установщика для развертывания новой версии.
1)     Запустите командную консоль CMD: Пуск – Выполнить – набрать cmd и нажать Enter
2)     Перейти в каталог, где находится установщик. Если на Рабочем столе, то команда
cd "%USERPROFILE%\Desktop"
3)     Запустить установку агента:
Dpa.agent(#49).install.exe.
Важный момент: установочный файл агента можно скачать с DPA Host. Для этого заходим в веб-интерфейс, нажимаем «шестеренку» и выбираем пункт «Установочные файлы» (setup files в англ. версии). Тогда установочный файл будет называться dpa.agent.4.6.21.exe, где 4.6.21 это версия установленного DPA host.
 
 
 
 
 
В качестве дополнительной проверки запустите Task Manager (Диспетчер задач). В списке Процессов будет dpa.agent.exe
Важный момент: обратите внимание, что в списке Процессов так же присутствуют ncdde, mbdde, dhserver. Это службы ЧПУ, которые передают информацию о станке агенту DPA.
 
Так же в Службах появятся две новые записи:
Dpa.agent. Сам агент. Данная служба должна автоматически запускаться вместе с операционной системой HMI
Dpa.rs. Служба остановлена по умолчанию, поскольку используется в «особых случаях», когда не работают стандартные методы. Применяется для обновления агента и удаленного доступа.
 
 
Путь установки агента E:\Program Files\X-tensive\DPA
