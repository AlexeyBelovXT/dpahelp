Start errors

 
1. If requirements to preparation of the server were not observed, installation process cannot be complete correctly.
If in installation process there were errors, for example, connected with insufficiency of rights, the final screen will contain references to installation log files – send files to X-tensive.
                                                          setup wizard. stop install
 
If errors settings/failure of SQL work allowed system to start, but did not allow to be connected to the database at start of a system instead of a window of authorization the description of a problem will be provided. This screen needs to be sent to X-tensive.
                                                                                                     DB connection error
 
2. Before the address in It X-tensive will make sure that all requirements are observed, it is necessary to start SQL Server Management Studio to execute a number of checks.
For new databases:
· Whether "Setup of the database" was succeeded to create databases with names, elective you on a step?
· Whether it was succeeded to create tables?
For the existing databases:
· At security settings of databases there are accounts of the selected users?
 
  • 3. If there were problems with start of separate components of a system, start the application "Services" and check the status of the following services:
  • – DPA Server;
  • – DPA Host;
  • If one or several components are in stop status, start them in that order.
Before the address problem to X-tensive execute a number of checks:
  • Make sure that the user selected for start of services is added to the corresponding group of access (Local security policy \Local policy \user right assignment \login as service).
                                                                                                                                                  LocalSecurityPolicy (пример)
 
  • Start the command line on behalf of the administrator and make sure that the port selected for start of Host at the stage "DPA Host Configuration" is reserved for start from a user name (netsh http show urlacl):
                                                                                                         network parameters
 
4. If the database copy used by the DPAHost system are installed on one server, it is recommended to set dependence between services.
Start the application "Services" and copy the name of service SQL Sever, then call the console as administrator and execute a command "sc config "DpaHost" depend = "MSSQLSERVER".
