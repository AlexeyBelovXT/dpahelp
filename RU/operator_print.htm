Операции. Инициация запуска заказа
 
Действия доступные пользователю на данной вкладке пояснены ниже:
При отклонении от технологии, связанным с приостановкой заказа или превышение нормативного времени пользователь должен пояснить причину своего действия. Перечень причин может быть настроен в соответствующем справочнике («Причины отклонений» и «Причины приостановки операции»). Величина отклонений от нормы является настраиваемой (Безопасность: Учет простоев/Игнорирование коротких простоев в модуле оператор). 
Прогресс работы по операции визуализирован в виде таймера. Пока время выполнения укладывается в норматив таймер показывает прогресс. После превышения нормативного времени таймер меняет цвет на красный. 
 
После завершения партионной наладки система запрашивает у пользователя, готов ли он к новому циклу работы над изделием.
Модуль "Оператор". Заказы. Завершение операции
При завершении работы над очередной операцией цикла изготовления штуки, нажмите на кнопку - система просит подтвердить необходимость запуска следующей операции. 
 
Модуль "Оператор". Заказы. Следующая операция
В рамках операций категории «Ввод данных» необходимо заполнить требуемые поля и нажать .

При запуске операций категории «Работа по УП» система пытается определить имя УП активной в данный момент времени. Если УП не найдена/не совпадает с УП, из технологии можно загрузить нужную УП из хранилища.

По завершению операций цикла выпуска изделия предоставляется выбор: начать новый цикл или завершить работу по выпуску данной партии.
Модуль "Оператор". Заказы. Завершение операции
 
По завершению операций цикла выпуска изделия предоставляется возможность указать качество выпущенной единицы. 
Модуль "Оператор". Заказы. Корректировка количества изделий
 
По завершению операций партионной обработки (тип «завершающая-партия») система предлагает подтвердит завершение работы над заказом и общее число выпущенных изделий в разбивке по статусам годности. На этом этапе можно привлечь работников ОТК для корректировок предварительной оценки оператора.

Модуль "Оператор". Заказы. Корректировка предварительной оценки оператора
 
На странице «Простои» оператору предоставляется доступна функция классификация простоев недоступностей. Простои отображаются в двух режимах «Все» (в хронологическом порядке) и «Неклассифицированные». Если на сервере активирована опция «Игнорировать коротки простои в модуле оператор», простои менее установленной длительности не доступны для классификации. 
Модуль "Оператор". Статистика простоев
 
Для классификации простоя выберите событие, затем в открывшемся модальном окне – причину. 
Модуль "Оператор". Статистика простоев. Определение причины простоя
 
На странице «Сервис» оператор может формировать сервисные запросы в смежные службы. Запросы направляются на диспетчерский пульт для дальнейшей обработки.
Модуль "Оператор". Инициация формирования сервисных запросов
Статусная схема запросов:
  - Открыт;
- Отменен;
- В работе;
- Завершен;
- Подтвержден.
 
Примечание: Статус «Подтвержден» проставляет оператор по завершению работ специалистом в случае, если по мнению оператора запрос не выполнен, он возвращает его в работу.
