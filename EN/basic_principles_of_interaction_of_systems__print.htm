Basic principles of interaction of systems.

1. For each data type "master system" is selected (main).
 
The master system guarantees correctness and relevance of data of the corresponding type. Information "is created" and "edited" (changes) only in a master system.
Other systems carry out synchronization with a master system on the corresponding data type. Work with values strictly in the mode "only reading".
 
For each data type the master system can be selected:
The product range – ERP
Orders – ERP
Structure of a product – PLM
Managing programs – PLM
The operating plan – APS
Planned daily tasks – APS
The fact of execution of planned daily tasks – DPA
Machine data – DPA
etc.
 
2. Ways of integration
 
API, for example, web services or REST-API
Allows to organize interaction by the principle a request answer.
From pluses – data exchange in the mode close to real time.
From minuses – low fault tolerance. If one of systems is not available to time at present, requests will not be processed. It is required to provide mechanisms of repeated request processing in such cases.
 
Data buses, for example, MSMQ
Queues. Interaction by the principle "the first came - the first left".
There is a delivery guarantee, there is a determined message handling order. Can demand certain efforts of administration.
Gives good fault tolerance and scalability.
Other data buses are available, for example, RabbitMQ, NServiceBUS and others - can turn on the built-in gateway facilities, maintaining logs, diagnostics.
 
Integration databases
Gives a delivery guarantee. There is a tranzaktsionnost. It is possible to organize interaction of several information systems at once.
From minuses – development and programming of logic of work with the specific synchronizing database is required. Support and maintenance.
 
General file storage
File exchange. For example - XML. Provides a standard way of coding of contents, provides a flexible way of creation of data structures.
Does not guarantee a message handling order. But allows to organize integration between the systems of essentially different platforms (windows - and Linux system)
