Structure and data type.
DPA supports import of the XML files given through transfer. In a body of the file one or several tasks of import are described. Tasks can be classified in four main groups: only creation of an object, creation or updating of an object, only updating of an object, removal of an object. In addition to these groups of tasks, there are tasks for management of schedules of work and job controls.
The following entities can be imported to DPA:
  • Reference book "Crews"
  • Reference book "Type of cycles unitary enterprise"
  • Reference book "Idle Time Reasons"
  • Reference book "Types of the Reasons of Idle Time"
  • Reference book "Note Types"
  • Reference book "Reasons for rejection"
  • Reference book "Positions"
  • Reference book "Processing Types"
  • Reference book "Product Types"
  • Reference book "Type of the Managing Program"
  • Reference book "Names of Changes"
  • Reference book "Suspension Reasons"
  • Reference book "Groups of Workers of the Centers"
  • Reference book "Templates of Schedules"
  • Working schedules of the working centers
  • Managing programs and their versions
  • Products and their versions
  • Technologies
  • Planned daily tasks and management of them
 
Data type and the transferred values
  • Crew (the name, the foreman, the list of the personnel entering into crew)
  • Type of cycles unitary enterprise (name)
  • Idle time reason (name, color, idle time reason type, category, signs of availability to installation in analysts and operator)
  • Type of the reasons of idle time (name)
  • Type of the note (name)
  • Reasons for rejection (name, color)
  • Position (name)
  • Type of processing (name)
  • Type of a product (name)
  • Type of the program (name)
  • Name of change (name, color)
  • Suspension reason (name, color)
  • Group of workers of the centers (name, list of the equipment)
  • Schedule template (name, type, list of intervals)
  • Managing program (name, processing type, program type, category, division, comment)
  • Version of the managing program (status, version number, description, quantity of products, adjustment time, runtime, file, list of the equipment)
  • Product (code, name, category, type, division, comment)
  • Version of a product (status, version number, description, list of the equipment, list of components)
  • Technology (the name, the description, the activity period, the status, the made product, the list of stages, the list of media files)
  • Technology stage (name, description, list of operations, list of the equipment, list of media files)
  • Technological operation (name, description, duration, type, list of media files)
  • Planned daily tasks (the identifier from an external system, the description, the comment, the status, the equipment, planned and relevant time of the beginning/termination, the list of media files, the list of links to documentation)
  • Production (order, lot number, type, planned quantity, technology stage, version of a product)
  • Maintenance&Repairing (type, reason)
 
The complete information about structure of XML files can be obtained from the xsd-schemes placed on our ftp (ftp://ftp.x-tensive.com/Public/Downloads/DPA.Schemas/)
