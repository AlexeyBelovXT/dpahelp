Перенос компонентов системы на другой сервер

 
Перенос базы данных
Если переносу подлежит только база данных (переустановка компонентов не производится), выполните следующие действия:
·     Создайте резервную копию (backup) баз данных системы (например, X-tensive.Project109.Config и X-tensive.Project109.Host);
·     Создайте в новом экземпляре MS SQL базы данных с именами идентичными ранее существующим;
·     Выполните задачу восстановления баз из резервной копии (Правый клик по имени базы – Задачи/Восстановить/База данных…);
·     Отредактируйте строку подключения:
o Запустите config (http://localhost:81/);
o Деактивируйте и разблокируйте актуальный пакет;
o Откройте модальное окно и отредактируйте строку подключения: укажите имя нового сервера БД;
o Примените изменения;
o Выберите пункт «сгенерировать Snk-ключ»;
o Сохраните изменения;
o Последовательно выполните команды: «скомпилировать», «заблокировать», «upgrade».
 
Перенос базы данных и Project109
Если переносу подлежит только база данных и Project109, выполните следующие действия:
·     Создайте резервную копию (backup) баз данных системы (например, X-tensive.Project109.Config и X-tensive.Project109.Host);
·     Создайте в новом экземпляре MS SQL базы данных с именами идентичными ранее существующим;
·     Выполните задачу восстановления баз из резервной копии (Правый клик по имени базы – Задачи/Восстановить/База данных…);
·     На сервере DPA откройте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config  в строке вида <wcfTransport ipAddress="192.168.0.X" port="45618" />» укажите ip-адрес сервера, на который будет перенесен Project109;
·     Установите Project109 но новый сервер, на шаге «Настройка базы данных» выберите существующие базы.
 
Перенос Project109
Если переносу подлежит только Project109, выполните следующие действия:
·     Установите Project109 но новый сервер, на шаге «Настройка базы данных» выберите существующие базы;
·     На сервере DPA откройте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config  в строке вида <wcfTransport ipAddress="192.168.0.X" port="45618" />» укажите ip-адрес сервера, на который был перенесен Project109;
·     Подключите сервер к новому хосту.
 
Перенос сервера DPA
Если переносу подлежит DPA и Project109, выполните следующие действия:
 
·     Скопируйте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config;
·       Произведите установку DPA сервера;
·       Ранее скопированном файле DpaServer.config разместите в директории %\ProgramData\X-tensive\DPA\Config\DpaServer.config, заменив файл, автоматически созданный при установке;
·     Подключите новый сервер к хосту;
·       Запросите и установите новую лицензию.
 
Перенос базы данных, сервера DPA и Project109
Если переносу подлежит база данных, DPA и Project109, выполните следующие действия:
·     Создайте резервную копию (backup) баз данных системы (например, X-tensive.Project109.Config и X-tensive.Project109.Host);
·     Скопируйте файл %\ProgramData\X-tensive\DPA\Config\DpaServer.config;
·     Создайте в новом экземпляре MS SQL базы данных с именами идентичными ранее существующим;
·     Выполните задачу восстановления баз из резервной копии (Правый клик по имени базы – Задачи/Восстановить/База данных…);
·     Произведите установку DPA сервера;
·     В ранее скопированном файле DpaServer.config в строке вида <wcfTransport ipAddress="192.168.0.X" port="45618" />» укажите ip нового сервера, затем разместите файл в директории %\ProgramData\X-tensive\DPA\Config\DpaServer.config, заменив файл, автоматически созданный при установке;
·     Установите Project109, на шаге «Настройка базы данных» выбрать существующие базы;
·     Запросите и установите новую лицензию.
