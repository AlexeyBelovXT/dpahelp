Станок можно представить в качестве простой блок-схемы:
 
 
PCU – по сути ПК внутри станка. Обеспечивает пользовательский интерфейс для работы со станком специалистов разного уровня доступа. PCU бывают разных версий. Для них разные HMI
На PCU20 HMI Embedded, на PCU50, 70 HMI Advanced или Operate. Так же PCU отвечает за подключение других рабочих мест операторов (TCU)
NCU – Основной управляющий модуль. Контролирует передачу информации между узлами станка. Предоставляет информацию о состоянии различных индикаторов.
TCU – панель управления, с помощью которой оператор может управлять станком. Подключается либо через отдельный Eth-порт, либо через PCU.
Управляющая шина – среда передачи данных, которая обеспечивает обмен информацией между компонентами станка. Использует в качестве средства передачи данных особый протокол, например, Profibus. Так же для повышения автоматизации процессов может использоваться дополнительное ПО, например, STEP7.
 
На данный момент использование ПО Step7 позволяет считывать состояние станка напрямую, через интерфейс Ethernet.
Если система не поддерживает Step7, то требуется установка Агента DPA на PCU станка.
 
Перед подключением станка к DPA надо определить его модель, версию HMI и поддержку STEP7.
 
По умолчанию HMI загружается в режиме пользователя.
Для просмотра и изменения системных настроек необходимо зайти в режим администрирования.
